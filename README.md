# IDEA MAKERS

5W1H+メーカー、物語テーマメーカー、8つの創作メーカーを収録した静的Webアプリです。合計10メーカーの要素をランダムに組み合わせ、情報整理から物語制作まで支援します。

## 対応形式

- 5W1H
- 5W2H
- 5W3H
- 6W1H（Whom / Which）
- 6W2H（Whom / Which）
- 4W1H
- 5W1H ＋ So What
- 5W1H ＋ Now What
- 5W1H ＋ What if

## 使い方

データを `fetch` で読み込むため、`index.html` を直接開かず、ローカルサーバーまたはGitHub Pagesから表示してください。

```powershell
python -m http.server 8000
```

起動後、ブラウザーで `http://localhost:8000/` を開きます。

ジャンル・雰囲気・目的はチェックボックスで複数選択できます。複数選択時は、引き直すたびに各項目から最大2つを生成条件として採用し、What・Why・Howなど既存要素の内容へ反映します。テーマを独立した要素として追加しないため、5W1HならWho・What・When・Where・Why・Howの6要素を厳守します。

物語テーマメーカーは `http://localhost:8000/story.html` から利用できます。基本テーマ（6要素）、人物中心（8要素）、展開中心（10要素）、フル構想（14要素）を切り替えられます。30ジャンルは複数選択でき、選んだジャンル内で各要素をミックスします。未選択時は全ジャンルが対象です。

創作メーカー一覧は `http://localhost:8000/makers.html` から利用できます。次の8メーカーがあり、すべて基本6・発展8・詳細10・フル14要素、30ジャンル複数選択に対応します。

- キャラクターメーカー
- 世界観メーカー
- プロットメーカー
- シーンメーカー
- 関係性メーカー
- 台詞・会話メーカー
- 事件・謎メーカー
- タイトルメーカー

## データ

`data/` に次の14要素を分けて収録しています。各ファイルはUTF-8の文字列配列で3,000件ずつ、利用対象は計42,000件です。すべての要素を20文字以内に収めています。

- `who.json` — Who / だれが
- `whom.json` — Whom / だれに
- `what.json` — What / なにを・なにが
- `which.json` — Which / どれを
- `when.json` — When / いつ
- `where.json` — Where / どこで
- `why.json` — Why / なぜ
- `how.json` — How / どのように
- `how_many.json` — How many / いくつ・何人
- `how_much.json` — How much / いくら・どの程度
- `how_long.json` — How long / どのくらい
- `what_if.json` — What if / もし〜なら
- `so_what.json` — So what / それで何が言える
- `now_what.json` — Now what / 次に何をする

以前の文章生成で使っていた `action.json` も互換用データとして残しています。固有人物名、作品固有のキャラクター名、企業・ブランド・商品名を意図的に避け、家族、食事、買い物、散歩、趣味、天気など、日常的で一般的な語彙を中心にしています。

`facets.json` には、身近・多彩・少数派の3段階でジャンル68件、雰囲気40件、目的40件を収録しています。暮らしや家族などの身近な題材から、深海、古地図、製本、音の採集、思考実験などの少数派までを含みます。

`data/story/` には、主人公、舞台、時代、願い、欠落、きっかけ、目的、対立者、障害、関係、秘密、モチーフ、主題、結末の14要素を各3,000件、計42,000件収録しています。各項目は20文字以内で、`story-genres.json` の30ジャンルに各100件ずつ対応しています。固有作品やブランドに依存しない一般的な創作素材です。

`data/makers/` には8つの創作メーカーを分けて収録しています。各メーカーは14要素×各3,000件＝42,000件、8メーカー合計336,000件です。物語テーマと合わせて30ジャンルを共有し、各分類につき1ジャンル100件ずつ収録しています。5W1H・物語テーマ・創作メーカーの主要データ総数は420,000件です。

## 再生成と検証

```powershell
node scripts/generate-word-data.mjs
node scripts/validate-word-data.mjs
```

検証では、各3,000件の件数、20文字以内の上限、型、空文字、完全重複、NFKC正規化後の重複、Jaccard類似度0.80以上の候補、カテゴリ固有の語尾に加え、既存テーマ148件、物語30ジャンル、8メーカー×14分類、ジャンルごとの件数、HTMLとJavaScriptのDOM参照を確認します。

## 構成

- `index.html` — 画面構造と形式ガイド
- `story.html` — 物語テーマメーカーの画面構造
- `makers.html` — 8つの創作メーカー一覧
- `maker.html` — 創作メーカー共通画面
- `styles.css` — レスポンシブ表示と装飾
- `app.js` — 形式切替、テーマの複数選択とミックス、データ読込、抽選、個別再抽選、コピー
- `story.js` — 構想切替、ジャンルの複数選択、物語要素の抽選とコピー
- `maker.js` — 8メーカー共通の構成切替、ジャンル選択、抽選、コピー
- `data/` — 要素別の語彙データ
- `scripts/` — 語彙の生成・検証スクリプト

外部ライブラリやビルド工程は不要です。
